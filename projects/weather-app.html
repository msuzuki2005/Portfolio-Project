Weather App

A responsive weather interface that surfaces current conditions, hourly forecasts, and weekly outlooks with a calm, readable layout.

Client
ARTGR 4840/5840
Role
Designer & Developer
Tools
HTML · CSS · JavaScript
View live site
Hero image

Weather App was built to practice API integration and responsive UI design. The goal was a forecast experience that feels immediate and uncluttered — quick to scan before heading out the door.

Challenge

Most weather interfaces pack too much data above the fold. Users wanted current temperature and the next few hours at a glance, without hunting through tabs or ads.

Process

Define

Prioritized location search, current conditions, and hourly strip.

Ideate

Sketched mobile-first layouts with emphasis on typographic hierarchy.

Design System

Defined colors, typography, and reusable UI components for a consistent interface.

Prototyping

Built a working prototype connected to a weather API.

Sketching

Early sketches explored how to layer current weather, hourly trends, and a seven-day summary without overwhelming the viewport.

Reflection

Building Weather App reinforced how much clarity comes from restraint — showing less data, but showing it better, made the experience feel more trustworthy.

Next steps would include geolocation support and dynamic backgrounds that shift with conditions.

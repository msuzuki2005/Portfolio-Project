App Design

Caffeine Jitter Tracker

A playful health tracker that logs caffeine intake, visualizes daily totals, and helps users notice patterns before the jitters hit.

Client
ARTGR 4840/5840
Role
UI/UX Designer
Tools
Figma · JavaScript · Chart.js
View live site
Hero image

Caffeine Jitter Tracker started as a personal project to make sense of coffee habits during a busy semester. The app turns quick logs into a visual daily summary so users can spot overconsumption early.

Challenge

Caffeine tracking tools often feel clinical or guilt-driven. The goal was a friendly interface that encourages awareness without shame — fast logging with meaningful feedback.

Process

Define

Focused on one-tap logging and a clear daily caffeine meter.

Ideate

Explored chart styles and playful microcopy for high-intake warnings.

Design System

Defined colors, typography, and reusable UI components for a consistent interface.

Prototyping

Built an interactive chart dashboard with Chart.js.

Sketching

Sketches tested different ways to show running totals — radial meters, bar charts, and timeline views — before settling on a simple daily bar with hourly breakdown.

Reflection

This project showed how data visualization can feel approachable when paired with human copy and a restrained color system. The warning state uses both color and text — never color alone.

Future versions could add sleep correlation and customizable caffeine limits based on body weight.
